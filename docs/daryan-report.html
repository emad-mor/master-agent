The architecture

Layered memory

Daryan doesn't keep a flat transcript. Memory is four tiers; a turn ages left → right, getting more compressed at each step — verbatim turn, then a one-paragraph summary, then a one-line theme. Core is the only global tier; the other three are per project, so juggling many projects never bleeds one conversation into another.

On every Send

Daryan assembles the prompt's memory block, injects it (first turn, after an idle gap, or when the prompt cache expired — otherwise it leans on --resume), then runs eviction so each tier stays within its limit.

core+themes+summaries+recent→prompt
✂️

Lean injection

The verbatim Recent block was ~80% of per-turn weight. Now each recent turn is compacted at injection time (stored turns stay full, so on-reload history is intact): UI markers stripped, big code blocks truncated to a reference, length capped, prompts clipped.

♻️

Free eviction

The voice-first reply already carries a [[SUMMARY]] recap — a ready-made turn summary. Eviction reuses it as the Mid-tier summary instead of spawning a model call; only turns without one fall back.

Known trade-offs

Honest limits

  • Word-level voice highlight is time-interpolated (Kokoro emits no word timestamps) — it can drift; sentence-level is exact.
  • Voice-first home replies collapse to the summary once a stream finishes (full text shows while streaming).
  • Injected memory truncates code blocks — the agent re-reads files when it needs verbatim code.
Next

Where it goes

  • Shrink the verbatim window (20 → ~5 turns) — compounds with the compaction already shipped.
  • Tier core memory per project so cross-project work loads only what's relevant.
  • Validate memory hygiene over a few real multi-day sessions.
